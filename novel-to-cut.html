小说 → 成片粗剪,并把 Claude Code 装进 n8n

n8n 只装在 5090 服务器上也完全够用。本页教你搭一条「小说自动改编→分镜→生成→粗剪」的流水线,并让 Claude Code 作为“AI 编剧/导演”接入其中。

0先说清:n8n 在 5090 上,够用

n8n 是“指挥官”不是“劳工”,自己几乎不吃资源,活儿全靠网络调别的服务。它在哪台机器无所谓。

n8n 调用谁在哪怎么调
出图 / 配音 / 配乐 / 字幕 / ffmpeg 剪辑B 本地http://localhost:端口
视频生成(6×H100)、vLLM 大模型A 远程http://A的IP:端口
🧠 Claude Code(创作大脑)B 宿主机http://B宿主机IP:8787
前提只有两条:两台机器网络互通 + 共享同一个 /media(NFS)。你现在就能开工,不用等 A 上也装 n8n。

1工作流:小说 → 成片粗剪(10 步)

注意每步是 ☁️ 云端 Claude Code 还是 🖥️ 本地模型、跑在 A 还是 B。

#阶段谁来做位置
0小说输入(.txt 丢进 inbox)n8n 监听B
1小说 → 剧本改编☁️ Claude Code云
2剧本 → 分镜 project.json☁️ Claude Code云

4关键帧(逐镜)ComfyUIB(5090)

“粗剪”= 能判断故事成不成立的样片。第 9 步不做精细调色/转场,只按剧本顺序拼片、配人声+背景乐、烧字幕,输出 rough_cut.mp4。精修留人工——自动化最该产出的就是这个粗样片。

2Claude Code 的价值(不是花瓶)

关键:它不是“又一个 LLM 调用”,而是一个能用你的技能与记忆、能读写文件的智能体。所以让它负责 3 个高含金量环节:

  • 第 1 步 小说→剧本:读整本小说、调用你的 documentary-director 技能与项目记忆(《锦衣少年行》《故宫之下》设定),写出成篇剧本——本地 vLLM 单次调用做不到这种“长上下文+专属范式+系列连续性”。
  • 第 2 步 分镜:按导演手法切成带景别/运镜/情绪的镜头表。
  • 第 10 步 质检:读 project.json+质检报告,像导演一样判断哪个镜头不对、为什么、怎么重拍。
诚实提醒:Claude Code 走 Anthropic 云 API,不是本地模型。上云的只有文本(小说、剧本、分镜描述、质检意见);所有画面/视频/声音/成片永远在你服务器上。若小说文本不能外传,第 1/2/10 步改用本地 vLLM(下面的子流程支持一键切换)。

3把 Claude Code 装进 n8n

推荐:claude-bridge 微服务跑在 B 宿主机(非 Docker),n8n 从容器里用 HTTP 调它。这样 Claude Code 原生加载你 ~/.claude 里的技能和记忆。

claude-bridge.mjsAgent SDK 封装的 HTTP 服务 package.json依赖清单 README.md安装启动说明 SUB_script.json小说→剧本 · Claude/vLLM 可切换
1

配认证(在 B 宿主机)

从 platform.claude.com 获取 key,写进 ~/.bashrc
export ANTHROPIC_API_KEY="sk-ant-..."
2

装并启动 bridge

下载上面三个文件到同一目录,然后
npm install
npm i -g pm2
pm2 start claude-bridge.mjs && pm2 save

# 冒烟测试:应返回 result + session_id
curl -X POST localhost:8787/agent -H 'content-type: application/json' \
  -d '{"prompt":"说你好并列出你能看到的技能"}'
成功标志:返回的 result 里能提到你的 documentary-director 技能,说明技能+记忆已加载。
3

n8n 里调用

HTTP Request 节点 → POST http://B宿主机IP:8787/agent,body 传任务;响应里 result 是产出、session_id 存起来给下一集续接。三个创作环节的 body 示例:

① 小说 → 剧本
{
  "prompt": "读 /media/proj01/novel.txt,改编成横屏精品短剧剧本,写到 screenplay.md。保留关键反转。",
  "cwd": "/media/proj01",
  "allowedTools": ["Read","Write"],
  "systemAppend": "你是资深短剧编剧,遵循用户创作范式。"
}
③ 粗剪质检复核
{
  "prompt": "读 project.json 和 qc_report.json,像导演一样找出不合格镜头,把 shot_id+原因写到 redo.json。",
  "cwd": "/media/proj01",
  "allowedTools": ["Read","Write","Glob"]
}
快速验证:不搭服务,用 Execute Command 直调 CLI

n8n Execute Command 节点直接跑(注意 别加 --bare,否则会跳过技能和记忆):

前提:claude 已装进 n8n 容器 / 或在宿主机跑
claude -p "读 novel.txt 改编成剧本写到 screenplay.md" \
  --add-dir /media/proj01 \
  --allowedTools "Read,Write" \
  --permission-mode acceptEdits \
  --output-format json

输出 JSON 含 result / session_id / total_cost_usd,用 Code 节点 JSON.parse 取。要强制结构化输出加 --json-schema '{...}'(结果在 structured_output 字段)。

4系列圣经连续性(做剧集必用)

第 1 集改编时 bridge 返回 session_id,存进 n8n(或写进 project.json)。第 2 集请求带上 "resume": "那个session_id",Claude Code 就记得第 1 集的人物、伏笔、世界观,保证跨集连续——对《锦衣少年行》5 季架构是刚需。

5Claude Code / 本地 vLLM 一键切换

SUB_script.json 已内置一个开关:传入 engine: "claude" 走 Claude Code(创作强、文本上云),传 engine: "vllm" 走本地大模型(全本地)。两条分支输出统一归一化后写同一个 screenplay.md。导入后把 B_HOST_IP / A_HOST_IP 改成你的真实地址即可。

6分步怎么做

  1. B 宿主机配 ANTHROPIC_API_KEY
  2. 下载 bridge 三件套,npm install + pm2 start,curl 测通
  3. n8n 加 HTTP 节点调 bridge,先把“小说→剧本”跑通,去 /media 看有没有 screenplay.md
  4. 导入 SUB_script.json,串进主流水线
  5. 接上“剧本→分镜→(A 生视频)→本地配音配乐→ffmpeg 粗剪”
  6. 加 session_id 续接,做到逐集连续
  7. 接第 10 步质检复核,产出 redo.json 驱动重做
